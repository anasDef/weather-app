# Weather App

**Live Demo:** <https://anasdef.github.io/weather-app/>

## Description

A responsive, modern weather application built to satisfy the **Frontend Mentor – Weather App** challenge. The app fetches real‑time weather data based on the user's location (via the browser Geolocation API) or a manually entered city, and displays temperature, wind, precipitation, and a 7‑day forecast.

---

## User Features (Non‑technical)

- **Automatic location detection** – on first load the app asks for permission to use your device’s location and shows the weather for your city.
- **Manual city search** – you can type any city name to view its weather.
- **Unit toggle** – switch between Metric (°C, km/h, mm) and Imperial (°F, mph, in) systems.
- **Responsive design** – works beautifully on mobile, tablet, and desktop screens.
- **Accessible UI** – proper ARIA labels, keyboard navigation, and high‑contrast colour options.
- **7-day forecast** – shows the weather for the next 7 days.
- **24-hour forecast** – shows the weather for the next 24 hours.

---

## Technologies Used

- **HTML5** – semantic markup.
- **Sass** – custom layout, dark‑mode support, and subtle animations.
- **Vanilla JavaScript (ES6+)** – no frameworks, pure modules.
- **OpenStreetMap Nominatim API** – reverse‑geocoding for city/country names.
- **GeocodingOpenMeteo API** – to get the coordinats of the place the user wants to search for.
- **OpenMeteo API** – to get the weather data of the place the user wants to search for.
